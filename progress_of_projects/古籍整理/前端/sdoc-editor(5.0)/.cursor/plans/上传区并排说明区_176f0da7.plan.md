# upload.html 并排布局调整计划

## 目标

- 让“上传说明”与“点击或拖拽上传”在桌面端同一行并排。
- 小屏下自动改为上下堆叠，避免内容拥挤。
- 保持“已上传列表”继续占据剩余空间，缩略图排列稳定不重叠。

## 变更文件

- [C:\Users\25283\Desktop\古籍整理\前端\sdoc-editor(4.4)_ready_for_ai - 副本\upload.html](C:\Users\25283\Desktop\古籍整理\前端\sdoc-editor(4.4)_ready_for_ai - 副本\upload.html)

## 实施步骤

1. 在 `upload.html` 的样式中新增一个上半区容器（如 `.upload-top-row`）用于包裹 `.tips-section` 与 `.upload-area`，并设置为 `display: grid` 或 `display: flex` 横向布局。
2. 调整 `.tips-section` 与 `.upload-area` 的宽度占比（默认建议 `38% / 62%`，接近黄金分割视觉），统一高度策略与间距。
3. 新增响应式断点（如 `@media (max-width: 900px)`）将上半区改为单列堆叠。
4. 维持 `.upload-page` 的纵向布局和 `.uploaded-list { flex: 1; min-height: 0; }`，确保并排后列表自动填充下方空间。
5. 保留并复核 `.uploaded-grid` 与 `.uploaded-item` 的防重叠策略（稳定卡片高度、顺序流式换行、图片完整展示）。
6. 回归检查：桌面端并排、小屏堆叠、空列表/多图/PDF 多页下列表与缩略图表现正常。

## 验收标准

- 桌面端“上传说明”和“上传区”左右并排显示。
- 小屏端自动恢复上下结构，内容不挤压。
- “已上传列表”位置上移并填充剩余空间。
- 缩略图按网格顺序展示，无重叠。
