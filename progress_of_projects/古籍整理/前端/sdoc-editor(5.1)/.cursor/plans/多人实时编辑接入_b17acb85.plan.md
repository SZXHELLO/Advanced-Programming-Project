# 多人实时编辑接入计划

## 目标
- 在 `project.html` 导入项目时检测同项目在线编辑人数；存在其他在线用户则进入多人实时编辑模式。
- 在 `edit.html` 底部状态栏“缩放”右侧显示：`开启多人编辑：${在线人数}`。
- 同一项目的在线用户之间实时同步新增方框与右侧卡片。
- 当他人正在拉框时，在其框下方显示昵称浮框（宽度自适应昵称单行显示）。
- 在线判定采用你指定规则：**手动退出才离线**。

## 实施步骤
1. **后端 WebSocket 协同房间（核心）**
   - 在 `backend/server.js` 将启动方式从 `app.listen` 改为 `http.createServer(app)` + `WebSocketServer`。
   - 新增依赖 `ws`。
   - 维护内存结构：`projectRooms: Map<projectId, Map<sessionId, clientMeta>>`。
   - WebSocket 握手携带 `sessionId + projectId`，复用现有会话验证。
   - 协议事件（最小集）：
     - `join`：加入项目房间，广播在线人数与用户列表；
     - `drawing_preview`：广播“他人正在拉框”的临时矩形与昵称；
     - `annotation_add`：广播新增标注（含 pageId、rect、attrType、note、color 等）；
     - `leave`：用户手动退出协同，更新在线人数。
   - 说明：按你的规则，**仅收到 `leave` 才判离线**；页面关闭/断线可先保留会话，不从在线列表移除（后续可再加清理策略）。

2. **导入页多人模式检测与标记**
   - 在 `project.html` 的 `importSelectedProject()` 中，导入项目后跳转前调用新增 REST 接口（如 `GET /api/collab/status?projectId=...`）查询当前在线人数。
   - 若在线人数 > 1（含自己接入后），将多人模式标记写入 `sessionStorage`（如 `collab_enabled=true`、`collab_project_id`）。
   - 保持跳转 `edit.html`，由编辑页读取该标记并建立 WS 连接。

3. **编辑页状态栏提示与连接管理**
   - 在 `edit.html` 底部状态栏“缩放”右侧新增协同状态节点（如 `collabStatus`）。
   - 初始化时读取 `sessionStorage` 与 `AppState.project.id`，若开启协同则连接 WS。
   - 监听在线人数广播并渲染：`开启多人编辑：${onlineCount}`。
   - 新增“手动退出多人编辑”动作（按钮或菜单），触发 `leave` 后更新提示并关闭 WS。

4. **标注实时同步（新增方框 + 右侧卡片）**
   - 本地新增标注时（`createAnnotation` 成功后）发送 `annotation_add`。
   - 收到他人 `annotation_add` 时：
     - 去重（按 `id` 或 `originSessionId + timestamp`）；
     - 写入 `AppState.annotations`；
     - 调用 `redrawAnnotations()` 与 `updateAnnotationList()`，确保 canvas 与右侧卡片同步展示。

5. **他人拉框昵称浮框**
   - 本地拉框过程中（`handleCanvasMouseMove`）节流发送 `drawing_preview`（包含 rect + displayName）。
   - 远端接收后在交互层绘制临时框，并在框下方渲染昵称标签 DOM/Canvas。
   - 标签样式：`white-space: nowrap` + 自适应 padding，保证单行显示完整昵称。
   - 拉框结束（`mouseup`）发送清除预览事件，隐藏该用户临时框与昵称标签。

6. **回归与验收**
   - A、B 两个账号导入同一项目时，编辑页状态栏在线人数实时更新。
   - A 新增方框后，B 的 canvas 与 annotation-list 同步出现。
   - B 拉框过程中，A 可见该临时框与昵称浮框。
   - 点击“退出多人编辑”后，该用户不再收到协同事件，在线人数按规则更新。

## 风险与约束
- 你选择“手动退出才离线”会导致异常关闭页面时仍可能显示在线（这是规则使然）。
- 当前先做“新增方框/新增卡片”同步，不包含复杂冲突合并（删改冲突后续可增量扩展）。
